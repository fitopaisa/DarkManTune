Ramas y fusiones

Rama master

Se genera por defecto cuando hacemos git init.
donde se genera el primer commit master

git branch

Nos muestra las ramas.

Si ponemos branch text, donde text el el nombre que definimos a esta rama.
creamos y nos vamos a la rama text.
ahora los cambias que hagamos se haran el la rama text y no en master.
por tanto ahora tenemos dos ramas , master y texto y estamos en activo text.

Para probarlos hacemos algun cambio y
git add -A // guardamos los cambios git commit -m "El comentario que quieras : realizados cambios en la rama text"

Fusiones

Si las pruebas de la rama text las queremos guardar en el Master usaremos las fusiones

Nos vamos a la rama que va a absorber los cambios de text
y lo hacemos con git checkoutpara movernos y git merge para fusinarla

git checkout master
git merge text

Tras fusionar con merge la rama text si cambiamos con checkout a text veremos qeu tienen los dos ya el mismo codigo. hemos fusionado los cambios. y actualizado la rama master.

cuando ya tengamos cambios en una rama text y actualizamos la raiz master.
podemos volver a la rama texto hacer merge de raiz para tener la ultima version de raiz master y cuando estemos contentos con los cambios volvemos a master y hacemos fusion merge de la raiz text.

Hay dos tipos de fsion

fast-forward - Simple y automatico.
manual Merge - Largo y Manual.

Para probar los tipos de fusion vamos a crear mas ramas no solo la de text.
git branch Colores
creamos un archivo css para los colores y los cambios que deses XD.

git add -A
git commit -m "Cambio de colores en rama colores"

ahora si cambiamos de ramas master y colores veremoe que los cambios estan solo en la rama de colore.
En master no hay cambios hata que decidadmos hacer fusion.

git checkout master
git checkout colores

Si no te gustaron lso cambos o no te gusta la version que hemos echo en la rama colores pues borramos la rama
git checkout master
git branch -D colores

git branch

Y queda borrada la rama sin que sufra ningun cambio master.
